What is the WWW

The main components of the web are:

  • The internet: HTML documents stored in servers around the world are reacheble through the world largest computer network

The HTML (Hyper Text Markup Language)

W3Schools definition: HTML is a markup language for describing web documents (web pages). HTML stands for Hyper Text Markup Language. A markup language is a set of markup tags. HTML documents are described by HTML tags. Each HTML tag describes different document content.

The HTML is composed of:

  • Contents: what appears on a browser
  • Markups: it defines how the content will appear
  • References to other docs: like immages or videos
  • Links to other pages

The markups are:

  • Tag: An HTML tag is always contained within angled brackets. Most of the tags have an opening and a closing tag. Some tags called void do not require a closing tag.
  • Element: an HTML element refers to everything within a set of opening and closing tags
  • Attribute: This is a property of an HTML element

HTML elements can be:

  • Inline: <span>
  • Block: <div>

The Document Object Model (DOM)

From Wikipedia: The Document Object Model (DOM) is a cross-platform and language-independent convention for representing and interacting with objects in HTML, XHTML, and XML documents. The nodes of every document are organized in a tree structure, called the DOM tree. The DOM is an application programming interface (API) for valid HTML and well-formed XML documents. It defines the logical structure of documents and the way a document is accessed and manipulated.

Adding CSS style and HTML structure

CSS or Cascading Style Sheets provide the style to our HTML documents. CSS is code written to control the "style" of HTML elements. What is very important is avoid repetitions between the two docs.

"Cascading" means that rules are applied not only to the elements they directly match, but also to all of those elements' child elements. However, if a child element has multiple, overlapping rules defined for it, the more specific rule takes effect.

Cascading is known as the idea of inheritance which is the mechanism by which properties are applied not only to a specified element, but also to its descendants.

Selector

Defines which elements of a page a particular style will apply to. A Selector is the first thing you write. A style can be written for a specific set of selectors (paragraph, class, h, IDs, etc). Everything between these curly braces will be interpreted as a "rule" that should be applied to every selector element.

The box model

  • Content: this is the image or text appearing on the website. It sits at the center of the box and the other layers serves to protect the content so that other images or text wont over lap
  • Padding: clears an area around the content, it is affected by the background color of the box. Space inside the box that protects the content
  • Border: the goes around the content and padding. Its color is affected by the color propriety of the box
  • Margin: this clear an area around the border and it does not have a color it is completely transparent. A sort of the space around the boxes

The size of your actual element = the size of content+padding+border+margin. This makes a bit difficult to understand how much space the box will actually take in the browsers. An easier way is to use in CSS the rule

 box-sizing:border box;

Box positioning

  • Divs are block elements (as opposed to inline), so by default they take up the entire width of a page.
  • Adding the rule 
    display:flex;
    to the appropriate CSS will override this behavior and let divs appear next to each other.

Basic terminology:

  • Computer: a machine that needs a program to work and do computations. Computers can do millions of computations in seconds.
  • Program: A conputer can not do anything without a program. A program are instructions to a computer on what to do precise sequence of steps
  • Computation: is any type of calculation that follows a well defined model understood and expressed as, for example, algorithim, or a protocol.
  • High-level language: Python is an high-level language. This means that we can use it to write our programs that will be interpreted and executed by another program which is a Python interpretor that will translate the program in a langauge that the computer can understand directly. So instead of having to write a program that runs directly on a computer, the program we will write will be an input to the Python program which runs on the computer.

    • "High-level language" refers to the higher level of abstraction from machine language. Rather than dealing with registers, memory addresses and call stacks, high-level languages deal with variables, arrays, objects, complex arithmetic or boolean expressions, subroutines and functions, loops, threads, locks, and other abstract computer science concepts, with a focus on usability over optimal program efficiency. Unlike low-level assembly languages, high-level languages have few, if any, language elements that translate directly into a machine's native opcodes. Other features, such as string handling routines, object-oriented language features, and file input/output, may also be present. Wikipedia

  • Input: litteraly, to put data into a computer
  • Interpreter: An Interpreter is a program that implements or simulates a virtual machine using the base set of instructions of a programming language as its machine language. You can also think of an Interpreter as a program that implements a library containing the implementation of the basic instruction set of a programming language in machine language. An Interpreter reads the statements of a program, analyzes them and then executes them on the virtual machine by calling the corresponding instructions of the library.
